# ChargingTerminal 项目说明

## 1. 项目目标

这是一个面向 Linux 的充电柜应用层项目，重点是：
- 通过 Modbus RTU/RS485 与设备通信
- 采集温度、湿度等传感器数据
- 以后扩展到电压、电流、状态管理和 UI

## 2. 目录结构

- bin/：可执行文件输出目录
- src/：源码目录
  - device/：设备驱动与总线封装
  - sensors/：传感器模块
  - app/：业务逻辑与状态机
  - main.c：程序入口
- include/：头文件目录
- lib/：第三方库或内部库
- conf/：配置文件目录
- data/：数据文件目录
- tests/：测试代码目录
- docs/：文档目录
- build/：构建输出目录
- tools/：工具脚本目录

## 3. 当前已完成的模块设计

### 3.1 Modbus 总线模块

- 文件：src/device/modbus_bus.c
- 作用：封装 Modbus RTU 初始化、从机地址设置、上下文获取等逻辑

### 3.2 温度传感模块

- 文件：src/device/temperature_sensor.c
- 作用：通过 Modbus 读取温度和湿度数据

### 3.3 配置文件

- 文件：config/device.conf
- 作用：存放串口、波特率、从机地址等参数

## 4. 代码设计原则

- 设备通信相关代码放在 device/
- 传感器逻辑放在 sensors/ 或 device/ 中
- 主程序入口放在 main.c 或 tests 中用于验证
- 寄存器地址、从机地址等参数目前先写死，后续可迁移到配置文件中
